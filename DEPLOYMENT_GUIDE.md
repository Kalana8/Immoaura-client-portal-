# Deployment Guide: Immoaura Client Portal on Hostinger

## Prerequisites

- Node.js 18+ installed locally
- npm or yarn package manager
- Hostinger account with hosting plan
- Supabase project (already configured with database and Edge Functions)

---

## Step 1: Build the Project Locally

### 1.1 Install Dependencies
```bash
cd /path/to/client\ portal
npm install
```

### 1.2 Create Environment Variables
Create a `.env.production` file in the project root:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
VITE_ADMIN_EMAIL=admin@immoaura.com
```

**Get these from your Supabase dashboard:**
1. Go to Supabase Console → Settings → API
2. Copy the "Project URL" and "anon public" key

### 1.3 Build the Production Bundle
```bash
npm run build
```

This creates a `dist/` folder with optimized production files.

### 1.4 Test the Build Locally (Optional)
```bash
npm run preview
```
Visit `http://localhost:4173` to verify everything works.

## Step 2: Prepare Files for Upload

### 2.1 What to Include in the Zip

```
dist/
  ├── index.html
  ├── assets/
  │   ├── *.js
  │   ├── *.css
  │   └── images/
  ├── favicon.ico
  └── robots.txt

supabase/
  ├── functions/
  │   └── send-email/
  │       └── index.ts
  └── migrations/
      ├── 20251022041141_c8359f9d-763e-4e80-bb08-0ca78d8cecbe.sql
      ├── 20251022050000_fix_users_rls.sql
      └── 20251023000000_create_order_number_function.sql

.htaccess (see section 2.2)
```

### 2.2 Create `.htaccess` for SPA Routing

Create a `.htaccess` file in the root (same level as `dist/`):

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  
  # Don't rewrite files or directories
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  
  # Rewrite everything else to index.html for SPA routing
  RewriteRule ^ index.html [QSA,L]
  
  # Set proper cache headers for static assets
  <FilesMatch "\.js$|\.css$|\.woff2?$|\.png$|\.jpg$|\.gif$|\.svg$">
    Header set Cache-Control "public, max-age=31536000, immutable"
  </FilesMatch>
  
  <FilesMatch "\.html$">
    Header set Cache-Control "public, max-age=3600, must-revalidate"
  </FilesMatch>
</IfModule>
```

### 2.3 Create the Zip File

On macOS/Linux:
```bash
# From the project root, create the zip
zip -r immoaura-client-portal.zip dist/ supabase/ .htaccess -x "node_modules/*" ".git/*"
```

On Windows (PowerShell):
```powershell
Compress-Archive -Path @("dist", "supabase", ".htaccess") -DestinationPath immoaura-client-portal.zip
```

Or use a GUI tool (7-Zip, WinRAR, etc.) to compress these folders.

## Step 3: Upload to Hostinger

### 3.1 Access Hostinger File Manager

1. Log in to **Hostinger Control Panel** (hPanel)
2. Go to **Files** → **File Manager**
3. Navigate to the **public_html** folder (this is your root directory)

### 3.2 Upload the Zip File

1. Click **Upload** button
2. Select `immoaura-client-portal.zip`
3. Wait for upload to complete

### 3.3 Extract the Zip

1. Right-click the uploaded `immoaura-client-portal.zip`
2. Select **Extract**
3. Choose destination: **public_html** (root)
4. Wait for extraction to complete

### 3.4 Verify File Structure

After extraction, your public_html should contain:
```
public_html/
├── dist/
│   ├── index.html
│   ├── assets/
│   ├── favicon.ico
│   └── robots.txt
├── supabase/
├── .htaccess
└── immoaura-client-portal.zip (delete this)
```

## Step 4: Configure the Hosting

### 4.1 Set Correct Permissions

1. In File Manager, right-click the `dist` folder
2. Select **Change Permissions**
3. Set to `755` for directories

### 4.2 Update the Index File

Most hosting providers use `index.html` by default, but you can verify in **Settings** → **General** that the document root is set to `/public_html/dist` or ensure `index.html` in the root works.

**Option A (Recommended)**: Move contents of `dist/` directly to `public_html/`
- Cut all files from `dist/`
- Paste them into `public_html/`
- Delete empty `dist/` folder

**Option B**: Keep `dist/` and configure document root in Hostinger settings

### 4.3 Enable HTTPS

1. Go to **SSL/Security** in hPanel
2. Click **Manage** under SSL certificates
3. Select **Autogenerated by Hostinger** (free)
4. Enable HTTPS redirect in **Settings** → **General**

---

## Step 5: Environment Configuration

### 5.1 Add Environment Variables to Hostinger

Since `.env` files won't work with static hosting, add your Supabase keys in one of these ways:

**Option A: Hardcode in Build** (not recommended for production)
- Update `.env.production` before building
- Rebuild and reupload

**Option B: Use Supabase Client Library** (current implementation)
- The client uses `import.meta.env.VITE_SUPABASE_URL`
- Add to `.env.production` before build
- Rebuild: `npm run build`

**Option C: Server-side Configuration** (best practice, requires Node.js)
- Contact Hostinger support to enable Node.js
- Deploy as a Node.js app instead

### 5.2 For Email Notifications

The Supabase Edge Function requires:
- **RESEND_API_KEY**: Your Resend API key
- **FROM_EMAIL**: Sender email address

Deploy the function:
```bash
supabase functions deploy send-email --project-ref your-project-id
```

Then set secrets in Supabase:
```bash
supabase secrets set RESEND_API_KEY=your-key-here FROM_EMAIL=noreply@yourdomain.com
```

## Step 6: Test the Deployment

1. Visit your domain: `https://yourdomain.com`
2. Test login functionality
3. Verify Supabase connection by checking console (DevTools → Console)
4. Check mobile responsiveness (visit on phone)

### Common Issues & Solutions

| Issue | Solution |
|---|---|
| Blank white page | Check browser console (F12) for errors; verify Supabase URL and keys are correct |
| 404 on page refresh | Ensure `.htaccess` is in `public_html` and rewrites are enabled |
| CORS errors | Check Supabase project settings for allowed origins |
| No images | Verify image paths in `src/` use correct relative paths |
| Email not sending | Confirm Edge Function is deployed; check Resend API key |

## Updating the Site

When you make changes:

1. Make changes locally
2. Run `npm run build` again
3. Zip the new `dist/` folder
4. Upload and extract in Hostinger File Manager
5. Clear browser cache (Ctrl+Shift+Del or Cmd+Shift+Del)

## Database Migrations

The Supabase migrations are already in place. To apply them to production:

```bash
supabase migrate up --project-ref your-project-id
```

Or manually run the SQL in Supabase SQL Editor:
1. Go to Supabase Dashboard → SQL Editor
2. Copy contents of migration files
3. Run each one in order
