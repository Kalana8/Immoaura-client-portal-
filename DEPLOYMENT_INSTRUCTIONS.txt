QUICK START - 5 MINUTES
═══════════════════════════════════════════════════════════════════════════════

1. LOG IN TO HOSTINGER
   → https://hpanel.hostinger.com

2. UPLOAD ZIP FILE
   → Files → File Manager → public_html
   → Click "Upload" → Select immoaura-client-portal.zip
   → Wait for upload to complete

3. EXTRACT FILES
   → Right-click immoaura-client-portal.zip
   → Select "Extract"
   → Destination: public_html
   → Click "Extract"
   → Delete the .zip file

4. CONFIGURE HOSTING
   → Settings → General
   → Document Root: /public_html
   → Enable HTTPS & Auto SSL
   → Save changes

5. ADD SUPABASE CREDENTIALS
   BEFORE UPLOAD, edit these in your environment:
   - VITE_SUPABASE_URL: Your Supabase project URL
   - VITE_SUPABASE_ANON_KEY: Your Supabase anon key
   - VITE_ADMIN_EMAIL: Your admin email

   These are already built into the production bundle.
   Get them from: Supabase Dashboard → Settings → API

6. TEST YOUR SITE
   → Visit your domain: https://yourdomain.com
   → Check console (F12) for errors
   → Test login functionality
   → Verify mobile responsiveness

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

❌ BLANK WHITE PAGE
   → Check browser console (F12 → Console)
   → Verify Supabase URL and keys are correct
   → Check .htaccess is in public_html

❌ 404 ERRORS ON PAGE REFRESH
   → Ensure .htaccess file exists in public_html
   → Check mod_rewrite is enabled in Hostinger

❌ CORS ERRORS IN CONSOLE
   → Go to Supabase Dashboard → Auth → Redirect URLs
   → Add your domain to the allowed list

❌ IMAGES NOT SHOWING
   → Verify dist/assets/ contains image files
   → Check image paths in console for 404s

❌ EMAILS NOT SENDING
   → Deploy Supabase Edge Function: 
     supabase functions deploy send-email
   → Set Resend API key in Supabase secrets
