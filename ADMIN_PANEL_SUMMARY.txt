🔐 KEY FEATURES FROM ADMIN BRIEF
═══════════════════════════════════════════════════════════════════════════════

Orders Workflow:
  • In Review → Confirmed → Planned → Delivered → Completed
  • Status changes auto-trigger emails to client
  • Admin can upload deliverables and message clients
  • Price breakdown always visible

Calendar Slots (for Video orders):
  • Lock slots when order status = Planned
  • No double booking allowed
  • System-wide availability management

Invoices (Manual, created by admin):
  • Auto-generates INV-YYYY-000001 format
  • Pre-fills from order data
  • Admin uploads PDF (required)
  • Payment status: Unpaid / Partial / Paid
  • Auto-completes order when invoice paid + order delivered

Email Notifications:
  • Order submitted (In Review) → client + admin
  • Order confirmed → client
  • Order planned → client (with slot info + 24h reminder)
  • Order delivered → client (with download links + invoice reminder)
  • Invoice issued → client
  • Invoice overdue → client

🔑 KEY REQUIREMENTS FROM BRIEF
═══════════════════════════════════════════════════════════════════════════════

1. Order Numbers: IM-000001... (FIXED with sequence - no more race conditions!)
2. Invoice Numbers: INV-YYYY-000001... (sequential, year-scoped)

4. Slot Locking: When order = Planned, slots become system-wide locked (no double booking)
5. Auto-actions: Invoice Paid + Order Delivered → Set Order = Completed

7. Calendar View: 12-month global availability, blackout/open toggle
8. PDF Upload: Required for invoice "Issued" status
