❓ YOUR QUESTION:
   "After refresh on /dashboard page shows 404, but works in new tab?"

✅ ANSWER: This is a COMMON issue with Single Page Applications (SPAs)

───────────────────────────────────────────────────────────────────────────────
🔍 THE PROBLEM:
───────────────────────────────────────────────────────────────────────────────

When you refresh at /dashboard:
1. Browser requests /dashboard from server
2. Server looks for physical file called "dashboard"
3. It doesn't exist → 404 Error

When you click a link or open in new tab:
1. Browser requests /index.html
2. Server finds it and serves it
3. React loads and Router handles /dashboard
4. It works!

───────────────────────────────────────────────────────────────────────────────
✅ THE SOLUTION:
───────────────────────────────────────────────────────────────────────────────

I've created a .htaccess file that tells the server:
"If a file doesn't exist, serve index.html instead"

Then React Router handles the routing on the client side.
