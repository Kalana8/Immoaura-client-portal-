🔴 ISSUE
═══════════════════════════════════════════════════════════════════════════════
Error: "duplicate key value violates unique constraint 'orders_order_number_key'"
Happens: When creating new orders (especially multiple simultaneous orders)

✅ ROOT CAUSE IDENTIFIED
═══════════════════════════════════════════════════════════════════════════════
Race condition in order number generation:
- The MAX() function is NOT atomic
- When 2+ users create orders simultaneously, both get the same order number
- First insert succeeds, second fails with constraint violation

✅ FIX CREATED
═══════════════════════════════════════════════════════════════════════════════
New migration file:
  📄 supabase/migrations/20251101192740_fix_order_number_sequence.sql

What it does:
  ✓ Creates PostgreSQL SEQUENCE objects (atomic, guaranteed unique)
  ✓ Replaces MAX() with nextval() for order number generation
  ✓ Maintains existing format: IM-000001, IM-000002, etc.
  ✓ Also fixes invoice number generation with same approach
