⚙️  IMPORTANT CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════════

Before uploading, you need to set your Supabase credentials.
These are built into the production bundle.

Get these from Supabase Dashboard → Settings → API:
  • VITE_SUPABASE_URL
  • VITE_SUPABASE_ANON_KEY

To rebuild with your credentials:
  1. Edit .env.production with your Supabase keys
  2. Run: npm run build
  3. Create new zip: zip -r immoaura-client-portal.zip dist/ supabase/ .htaccess
  4. Upload the new zip to Hostinger

OR you can hardcode them before the initial build if needed.

🔍 TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════════

Blank white page?
  → Press F12, check Console tab for errors
  → Verify Supabase credentials in browser
  → Check .htaccess file exists in public_html

404 on page refresh?
  → Ensure .htaccess is in public_html root
  → Contact Hostinger support to enable mod_rewrite if needed

CORS error?
  → Add your domain to Supabase → Auth → Redirect URLs

Emails not sending?
  → Deploy Edge Function: supabase functions deploy send-email
  → Add Resend API key to Supabase secrets
